  4. BUSINESS LOGIC RULES
================================================================================

BNPL Interest rates:
  3 months  = 0%
  6 months  = 3%
  9 months  = 6%
  12 months = 12%

Down payment:         Mandatory 20% of total amount
Credit range:         100 DT – 5000 DT
Due date formula:     salary_day + 2 days (per installment)
Payment simulation:   90% success rate

Credit score:
  Default score:      600
  On-time payment:    +10 points
  Late payment:       -15 points
  Score range:        300 – 850

Default credit limit: 5000 DT

Card security:
  Algorithm:          AES-256-GCM
  IV:                 Prepended to ciphertext, Base64 encoded
  CVV:                NEVER persisted
  Stored:             card_token (UUID), last4 (CHAR4), expiry_encrypted (AES)

Admin credentials:    admin@bnpl.com / admin123 (hardcoded dev)
Backend port:         8082
Expo port:            8081 (exp://192.168.1.167:8081)
Admin portal port:    3000


================================================================================
  5. USER JOURNEY (API CALL FLOW)
================================================================================

1. REGISTER
   POST /api/auth/register → { userId, email, firstName, lastName }
   POST /api/users/photo   → upload profile photo (optional)

2. LOGIN
   POST /api/auth/login → { userId, email, firstName, lastName }
   → store in AuthContext

3. KYC VERIFICATION (required before credit)
   POST /api/kyc/verify (cinFront, cinBack, selfie) → AI verify via Didit
   Status: NOT_SUBMITTED → PENDING → APPROVED / REJECTED

4. ADD PAYMENT CARD (required before credit)
   POST /api/cards/add → { cardNumber, expiryDate, type, isDefault }
   → card_number encrypted AES-256-GCM, CVV discarded immediately
   GET  /api/cards/user → list masked cards

5. FINANCIAL PROFILE (required before credit)
   POST /api/profile/create → { monthlySalary, salaryDay, employmentStatus }
   → risk level calculated automatically

6. CREDIT SIMULATION
   POST /api/credits/simulate → { amount, numberOfInstallments }
   ← { totalAmount, downPayment, monthlyAmount, totalInterest }

7. CREDIT REQUEST
   POST /api/credits/request
   → validates KYC approved + card exists + financial profile complete
   → generates installment schedule (due = salary_day + 2 days each month)
   → status: PENDING → (admin review) → APPROVED / REJECTED

8. PAYMENT
   POST /api/payments            (single installment)
   POST /api/payments/payAll     (all pending at once)
   → updates installment.status = PAID
   → updates CreadiScore (+10 on-time, -15 late)
   → sends PAYMENT_CONFIRMED notification

9. DASHBOARD
   GET /api/dashboard → { kycStatus, cardStatus, profileStatus, nextStep,
                          creditBalance, creditScore, activeLoans }
   nextStep values: COMPLETE_KYC | ADD_CARD | COMPLETE_FINANCIAL_PROFILE | READY_FOR_CREDIT
